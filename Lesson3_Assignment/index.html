Highlight #1 Version Control System

Version control systems help solve the problem with file sharing. They help people manage mutiple files at once and keeps track of who made chnages to files, when they were changed, and what was changed on the file. VCS's help so that mutiple people can work simultaneously without there being any conflcit between the mutiple workers. Files in the VCS are also not overwritten and backed up so you can revert back to past version of the file easily.

Highlight #2 Remote and Local Repository

There a a few importnat differences between the remote and local repository. The remote repository is stored on remote server where workers can work collaboratively on a project. The local repository would is stored on said workers local computers. After making chnages to the local repository it can then be uploaded to the remote repository for the whole team to see. In this class we will use Codio for our local repository and Github for our remote repository.

Highlight #3 Git Commands

In order to get info from our local repository to our remote repository we will use git commands to push it out to Github. The basic process to push our files to github will be git add ., git commit,git push. Git add will put files on our local staging area. Git commit will then add it to the local repository. Last Git push will then push it to the remote repository.
